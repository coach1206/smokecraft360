==================================================
5. REVENUE BRAIN LOGIC
==================================================

Create service:

server/services/revenueBrain.ts

Purpose:
Recommendations should not only match taste.
They should also help the venue make money without feeling manipulative.

Create:

scoreRevenueOpportunity(item, inventoryItem)

Factors:
margin
stock availability
vendor reliability
low stock penalty
premium item boost
popular tag boost

Formula:

tasteScore = tag match score from user memory
marginScore = inventory margin percentage
stockScore = stock > threshold ? 20 : -30
reliabilityScore = vendor reliability / 5
premiumBoost = price > average price ? 10 : 0

finalScore =
tasteScore * 0.45
+ marginScore * 0.25
+ stockScore * 0.15
+ reliabilityScore * 0.10
+ premiumBoost * 0.05

Return:
score
reason

Reason examples:
"Matches smoky and oak preferences with strong margin"
"High taste match but low stock, recommendation softened"
"Premium pairing selected because user shows bold-profile behavior"
