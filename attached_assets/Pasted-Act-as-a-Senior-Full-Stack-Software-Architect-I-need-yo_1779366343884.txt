### 2. CORE ENGINE & ROUTER LOGIC Rules
1. **Feature Flag Isolation:** Onboarding steps must NOT be bound strictly to the user's overall subscription tier status. The system must evaluate the individual feature boolean flags inside `featureFlags`.
2. **Deactivation Logic:** If a feature flag is set to `false`, its corresponding step status in `onboardingState.steps` must automatically resolve to "SKIPPED" or be ignored entirely by the layout router.
3. **Re-activation Logic:** If a feature flag is updated to `true` but its current onboarding step status is "SKIPPED", reset that specific step to "NOT_STARTED" so the user is dynamically prompted to complete that portion of the workflow.
4. **Clean Deterministic Output:** Return an ordered array of active, required onboarding steps that the frontend should actually render to the screen. Additionally, return a global boolean flag called `shouldShowOnboardingModal` (which resolves to `true` if any active feature flag has an onboarding step that is not yet marked as "COMPLETED").
