FINAL MASTER PROMPT

MISSING / REQUIRED

1. The current EEIE interface is not visually powerful enough.
Impact: It does not yet feel like a future-luxury Apple/Tesla hospitality intelligence system.

2. The interface must not be dark cyberpunk.
Impact: The correct direction is metallic blue, silver, white, titanium grey, pearl glass, chrome edges, frosted cards, premium images, and clean luxury spacing.

3. EEIE must not look like a normal POS.
Impact: EEIE means Experience Enhancement Intelligence Engine. The POS is only one embedded commerce layer inside a larger venue intelligence, staff assistance, guest experience, inventory, distributor, media, sensor, AI, and back-of-house system.

4. The system must be highly visual.
Impact: Staff need images, graphs, product visuals, AI commentary, pairing visuals, guest previews, heatmaps, sensor colors, mood states, and live visual guidance, not plain cards and text.

5. Staff need a real touchscreen workflow.
Impact: Wait staff must be able to select a table, pause the guest experience, see the guest’s preferences, show cigar/drink/food images, add items to cart, send to POS, and return the guest to the experience without losing progress.

6. Managers need a visual media control system.
Impact: Cigars, liquor bottles, cocktails, food, desserts, specials, merch, and pairing bundles must be uploaded or linked, approved, categorized, connected to inventory, and shown visually.

7. Back-of-house must connect to EEIE.
Impact: The system must connect to bar inventory, kitchen menu, cigar inventory, manager dashboard, staff cockpit, guest preview, commerce cart, POS adapter, event bus, and system health.

8. Distributor inventory libraries must be supported.
Impact: Cigar and liquor distributors may provide APIs, CSV files, spreadsheets, product links, PDFs, or manual lists. EEIE must support flexible import and mapping.

9. Distributor inventory must stay separate from venue inventory.
Impact: A distributor product is not automatically available at the venue. The system must separate distributor catalog, local stock, manager-only items, staff-visible items, guest-visible items, wholesale pricing, local pricing, restock suggestions, and special-order products.

10. AI assistant and AI commentary must be built into the interface.
Impact: EEIE must guide staff and managers in real time with recommendations, warnings, scripts, explanations, and next-best actions.

11. Mood and sensor intelligence must be included.
Impact: The system should show venue mood, service pressure, energy state, sensor readiness, haptics, vibration feedback, heatmaps, device signals, and live venue awareness.

12. All buttons must be functional.
Impact: Do not create dead UI. If backend routes exist, use them. If not, create frontend service wrappers that are ready for backend integration.
