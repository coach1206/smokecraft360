Build the complete Venue POS + Universal Experience Engine for SmokeCraft 360, PourCraft 360, VapeCraft 360, and BrewCraft 360.

This is the final operational layer. Do not leave placeholders, fake buttons, or disconnected UI.

The system must connect:

User selection → product recommendation → venue match → inventory availability → order / reservation → POS dashboard → commission tracking.

--------------------------------------------------
CORE PLATFORM STRUCTURE
--------------------------------------------------

The platform has four experience systems:

1. SmokeCraft 360
   Cigars, lounges, cigar pairings, private cigar sessions.

2. PourCraft 360
   Liquor, cocktails, whiskey, wine, bar pairings.

3. VapeCraft 360
   Vape flavors, device types, hit intensity, smoke/vapor preferences.

4. BrewCraft 360
   Beer styles, beer flights, brewery pairings, food pairings.

Each system must have:
- its own identity
- its own visuals
- its own questions
- its own products
- its own venue types
- its own recommendation logic
- but all must run on one universal engine.

--------------------------------------------------
UNIVERSAL EXPERIENCE FLOW
--------------------------------------------------

Every system must follow this flow:

1. User chooses system
2. User answers swipe-based questions
3. AI engine builds recommendation
4. System finds matching venue/bar/lounge/event
5. System checks inventory
6. User chooses:
   - Reserve experience
   - Add to cart
   - Pay now
   - Pay at venue
7. Venue receives order in POS dashboard
8. Commission is calculated
9. Order is logged
10. User receives confirmation

--------------------------------------------------
PAYMENT LOGIC
--------------------------------------------------

Support:

1. Pay now
   User pays through platform.
   Platform records payment.
   Commission is calculated.
   Venue payout is recorded.

2. Pay at venue
   User reserves.
   Venue confirms.
   Commission is tracked as pending.
   Venue marks as paid after in-person checkout.

3. Deposit
   User pays deposit.
   Remaining balance paid at venue.

4. Tab mode
   User adds products to a venue tab.
   Venue closes tab in POS.
