Build a premium, reusable Dynamic Visual Card Engine for a multi-experience platform called Craft Hub that powers four modules: SmokeCraft 360, PourCraft 360, BrewCraft 360, and VapeCraft 360. The goal is to completely replace all static images with a dynamic, rotating visual system that creates a high-end, cinematic, and immersive experience across all cards. Each card must automatically rotate through a curated set of high-quality images every 3.5 seconds using smooth fade transitions and subtle zoom animation, creating a living, breathing interface rather than a static UI. The system must be built in React with TypeScript and designed as a reusable component that accepts props for title, subtitle, and a structured scenes array, where each scene includes an image path and an optional label. The component must include a dark gradient overlay to ensure text readability, and display a title, subtitle, and a rotating scene label positioned in the top corner that updates in sync with the image. The animation must feel premium, using slow zoom (scale transform) and opacity transitions without jitter, and must be optimized for both desktop and mobile responsiveness with consistent aspect ratios.

SmokeCraft 360 must rotate through diverse human-centered imagery that reflects real lifestyle and inclusivity, including Black, Hispanic, White, male, female, professional, urban, and social environments, with scenes such as executive lounges, late-night smoking sessions, solo reflection, and group interactions, ensuring the experience feels authentic and culturally aware rather than staged. PourCraft 360 must rotate through a variety of drink experiences including whiskey neat, old fashioned cocktails, wine glasses, cocktail bar environments, and premium lounge drinks, emphasizing motion shots such as liquid pouring and glass detail to create richness and depth. BrewCraft 360 must showcase different beer types and textures including lagers, IPAs, stouts, beer flights, and fresh pours, focusing on foam, condensation, and variety to communicate choice and craft rather than a single static beer image. VapeCraft 360 must combine modern vape device visuals with hookah lounge environments, colorful vapor clouds, nightlife scenes, and social group interactions, ensuring the category feels like a real culture that blends technology, flavor, and lounge energy rather than abstract smoke visuals.

The system must use a centralized data structure that defines scene arrays for each module, with labeled entries such as “Executive Lounge,” “Craft Cocktail,” “Beer Flight,” and “Hookah Session,” and all image assets must be stored locally in a structured public directory under /images/smoke, /images/pour, /images/brew, and /images/vape. All images must be high-resolution, consistent in size, and visually cohesive in lighting and tone to avoid jarring transitions. The reusable DynamicCard component must handle the rotation logic internally using a timed interval that cycles through the scenes array, properly cleaning up intervals to prevent memory leaks, and must support smooth transitions using CSS animations including a keyframe zoom effect and opacity fade. The UI must avoid flat or generic design and instead feel like a premium, high-end system similar to luxury lounge or high-end hospitality interfaces, with layered depth, motion, and strong visual storytelling.

The final implementation must include the full React component code for DynamicCard, global CSS styling for animations, overlay, and layout, a centralized data file exporting all scene arrays for the four craft modules, and a main page implementation that renders all four cards in a responsive grid layout. The system must be production-ready, fully wired, and not rely on placeholder logic, with all image paths correctly referenced and no missing dependencies. Additionally, the design should be structured in a way that allows future integration with an AI recommendation engine, where image rotation can later adapt dynamically based on user behavior, preferences, time of day, or venue type, turning the visual system into an intelligent, responsive storytelling layer rather than a static rotation.
