Build a full-stack, production-ready platform called "SmokeCraft 360" that functions as a multi-industry experience and revenue system with offline capability, installable tablet support, multi-tenant architecture, and white-label customization.

TECH STACK:
- Backend: Node.js + Express
- Frontend: React (Vite)
- Database: PostgreSQL (with fallback to in-memory if unavailable)
- Authentication: JWT + bcrypt
- Styling: premium dark theme with gold accents
- State management: React hooks
- Storage: localStorage + IndexedDB for offline support

--------------------------------------------------
CORE SYSTEM REQUIREMENTS
--------------------------------------------------

1. RECOMMENDATION ENGINE (CORE INTELLIGENCE)

Build a modular engine supporting categories:
- cigar
- alcohol (whiskey, bourbon)
- food
- expandable for future categories

Each product must include:
- id, name, category
- flavorNotes (array)
- strength (1–5)
- moodTags
- pairingTags
- tier ("standard", "mid", "premium")
- boostLevel (0–3)
- sponsored (boolean)
- venueId

SCORING LOGIC:
- flavor match: +3 each
- mood match: +4
- strength:
  exact = +3
  within 1 = +2
  within 2 = +1
- tier:
  premium = +2
  mid = +1
- boostLevel = +1 per level
- sponsored = +2

NO negative scoring.

Return:
- top 3 recommendations
- top 2 alcohol pairings
- top 2 food pairings

--------------------------------------------------
2. PAIRING SYSTEM (SEMANTIC RULES)
--------------------------------------------------

Implement rules:

- strong/bold cigar → peaty/bold whiskey
- sweet cigar → bourbon/dessert pairing
- light cigar → light spirits + seafood/salad
- smoky → BBQ / grilled / wings

Apply reverse pairing for alcohol.
