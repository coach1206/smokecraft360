CARD ANIMATION REQUIREMENTS

Every EEIE module card must have:

1. Soft idle motion
A very subtle breathing glow around the border.

2. Hover lift
On hover or focus, the card lifts slightly and the metallic blue edge brightens.

3. Active scanline
A thin blue scanning line should pass across active cards every few seconds.

4. Image shimmer
Image preview strips should have a soft light pass across them.

5. Status pulse
Live, Active, Ready, Connected, Syncing, and Warning chips should pulse gently.

6. Press feedback
On tap/click, the card should compress slightly, glow brighter, then route.

7. Data flicker
Small status numbers should softly update or flicker like live telemetry, but not so much that it feels fake.

8. Route transition
When opening a module, fade the current hub out and slide the module in smoothly.

LIVING MACHINE PAGE EFFECTS

Add these global effects to the EEIE hub and EEIE pages:

1. Ambient machine glow
A subtle blue/cyan radial glow in the background that moves slowly.

2. Animated top waveform
The top rail waveform must move continuously.

3. Live telemetry dots
Small system dots should pulse at different intervals.

4. Event feed motion
New event items should slide in from the right or bottom.

5. Status rail movement
Bottom status rail should have a thin animated blue progress line.

7. AI assistant breathing glow
The AI Assistant panel should gently glow as if it is thinking.

8. Sensor rings
Sensor cards should have circular pulse rings to show signal strength.

9. Product wall motion
Product cards should slightly lift, image brighten, and show a soft “ready” shimmer.

10. Founder Control tension
Founder Control cards should stay mostly still, but danger cards should have a slow serious warning pulse.

ACCESSIBILITY REQUIREMENT

Respect reduced motion.

Add:

@media (prefers-reduced-motion: reduce)

When reduced motion is active:

turn off ambient movement
turn off scanlines
turn off shimmer
keep simple hover/focus states
keep status changes visible without motion

Do not remove usability for users who prefer reduced motion.

PERFORMANCE REQUIREMENT

Animations must be CSS-first.

Use transform and opacity when possible.

Avoid expensive layout animations.

Avoid animating width/height repeatedly.

Use will-change carefully only on cards that need it.

Do not create heavy animation loops in JavaScript.

Use requestAnimationFrame only if absolutely needed.
