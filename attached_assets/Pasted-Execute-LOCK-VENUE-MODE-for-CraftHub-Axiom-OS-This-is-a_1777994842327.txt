## 🧩 TASK 1 — SESSION CONTROL SYSTEM

Implement kiosk session isolation.

Requirements:
- Generate sessionId (UUID) per user session
- Track last interaction timestamp
- Reset session when:
  - 60 seconds inactivity
  - user completes reveal
  - user taps "Start Over"

resetSession must:
- clear all UI state
- clear cart
- clear selections
- generate new sessionId

---

## 🧩 TASK 2 — TOKEN REFRESH SYSTEM

Prevent token expiration failures.

Requirements:
- Detect expired token (401 responses)
- Automatically call POST /api/auth/kiosk
- Replace token
- Retry original request

Also:
- refresh token proactively every 30 minutes

---

## 🧩 TASK 3 — KIOSK IDLE RESET UI

Add recovery UI.

Requirements:
- Show overlay:
  "Session expired — Tap to restart"
- Include "Start New Experience" button
- Reset system completely on tap

---

## 🧩 TASK 4 — PAYMENT PROTECTION

Prevent duplicate payments.

Frontend:
- Disable checkout button after first tap
- Show loading state
- Block repeat taps

Backend:
- Require idempotencyKey per checkout request
- Reject duplicate requests with same key

---

## 🧩 TASK 5 — NETWORK RETRY SYSTEM

Handle unstable connections.

Requirements:
- Retry failed API requests (max 3 times)
- Use exponential backoff (500ms → 1s → 2s)
- Show retry UI after failure

---

## 🧩 TASK 6 — GLOBAL ERROR HANDLING

Requirements:
- Catch all API failures
- Show user-friendly error message
- Provide retry button
- Never fail silently

---

## 🧩 TASK 7 — MULTI-USER SAFETY

Requirements:
- Ensure no user data persists between sessions
- Clear:
  - selections
  - cart
  - visual state
  - session state

---

## 🧩 TASK 8 — PERFORMANCE GUARD

Requirements:
- Debounce rapid taps (300ms)
- Prevent duplicate triggers
- Throttle heavy actions
