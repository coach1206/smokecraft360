FINAL EEIE / SOVEREIGN GLOBAL INTERFACE REDESIGN PROMPT

Redesign the entire EEIE / Sovereign interface family so every page uses the same premium visual language as the lighter blue-and-white command reference.

This is the new global standard for ALL EEIE / Sovereign interfaces.

GOAL

The system should feel like:

Apple meets Tesla  
premium predictive intelligence  
3D visual command software  
a living and breathing machine  
futuristic, instinctive, clean, and executive  
not a generic POS system  
not a flat admin dashboard  
not too dark  
not too light  
high-end, visual, and operational

GLOBAL VISUAL DIRECTION

Use the light blue / white contrast style across all interfaces.

Design language:

deep navy top rail and selective deep navy side rail
soft white and light ice-blue content surfaces
metallic blue accents
titanium borders
electric cyan highlights
signal green for healthy/active
soft amber for attention
controlled red for alerts only
glow, glass depth, layered panels, soft shadowing
clean luxury whitespace
high legibility
premium motion
visual hierarchy

Do NOT use:

brown dominance
gold dominance
muddy backgrounds
flat blocks
solid dark blue everywhere
glyph-only tiles
emoji-like symbols
cheap generic fonts
plain developer tables with no productization

GLOBAL SYSTEM RULES

1. Every page must feel like part of the same interface family.
2. Every page must use the same color system, spacing system, and panel language.
3. Every page must have clear sections.
4. Every page must include useful visuals, not just data.
5. Every page must include subtle animation and living movement.
6. Every page must have strong readability.
7. Every major feature card must feel dimensional, not flat.

TYPOGRAPHY

Use clean premium modern typography.

Requirements:
page titles large and readable
section headers clearly distinct
body text clean and easy to scan
no tiny sci-fi microtext for important content
strong hierarchy
avoid cheap-looking condensed text
use crisp, refined, executive-style interface typography

GLOBAL LAYOUT SYSTEM

Every interface should use a structured layout with:

1. Top status/navigation bar
2. Left system rail or module rail
3. Main hero/overview band
4. Section-based content grid
5. Visual intelligence panels
6. Lower utility / ticker / live activity strip where appropriate

Use consistent rounded corners, premium panel depth, metallic borders, and subtle glow edges.

MASTER SECTION STYLING

Every section/card must feel like a purposeful operational module.

Cards should include:
clear header
subtext
visual content
operational stats
action buttons
status state
subtle motion

Do not use empty dead cards.

GLOBAL ANIMATION / LIVING MACHINE RULES

This is a living, breathing, learning system.

Add restrained premium animation across the full interface system:

top rail signal sweep
active tab glow pulse
card hover rise
soft glass shimmer
background signal waves
sensor pulse indicators
activity feed slide-ins
graph line animations
status ring pulses
heartbeat / breathing motion on active panels
live dots with subtle energy
ambient telemetry movement
module hover reveals
ticker movement where appropriate

Important:
Animations must feel intelligent and premium, not childish or gimmicky.

Add reduced-motion support:
If prefers-reduced-motion is enabled, simplify all motion.

NO GLYPHS / NO FLAT PLACEHOLDERS

Do not use generic glyphs as the main visual identity of tiles.
Do not rely on flat line icons as the main focus.
Use actual imagery, visual renderings, and dimensional visual elements.

SECTIONING REQUIREMENTS

visual breaks between sections

The user likes the blue-and-white contrast through the pages, not one solid blue block. Preserve that.
